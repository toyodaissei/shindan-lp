# セットアップ手順（初回テストまで）

所要時間の目安：**約15分**。順番どおりに進めれば、実データなしで3システム全部の動作確認までできます。

---

## ⚡ いちばん簡単な使い方（承認ボタン・Chat・Webアプリ すべて不要）

既定は `APPROVAL_MODE: false`（全自動）です。設定はこれだけ：

1. スクリプトプロパティに `GEMINI_API_KEY` と `REPORT_RECIPIENTS` を入れる（STEP 2）
2. `setup` を実行（シート・フォルダを自動作成）
3. `installTriggers` を実行（15分毎の自動処理＋シートのメニューをON）

**使い方 = Driveの「DM営業_画面録画」フォルダに録画/スクショを入れるだけ。**
15分以内に自動で提案書ができ、**スプレッドシートの「DM案件」シート**と**「AI生成_提案書」フォルダ**にリンクが出ます。

**誰でも“今すぐ実行”したい時** → スプレッドシートを開く → 上部メニュー
**「AI営業スイート」→「▶ 今すぐ全部まとめて実行」**（Apps Scriptを開く必要なし）。

> Google Chat通知や承認ボタンは“おまけ”です。要らなければ STEP 5・6.5 は飛ばしてOK。

---

## STEP 0. 準備するもの
- Googleアカウント（Workspace / 個人Gmailどちらでも可）
- **Gemini APIキー**（無料）… [Google AI Studio](https://aistudio.google.com/apikey) → 「Create API key」

---

## STEP 1. Apps Scriptプロジェクトを作る
1. [script.google.com](https://script.google.com/) → 「**新しいプロジェクト**」
2. 左の「ファイル」で、`ai-knowledge-system/gas/` の中の各ファイルを追加して中身を貼り付け：
   - `Config.gs` / `Llm.gs` / `Code.gs` / `Proposal.gs` / `DmCoach.gs` / `Setup.gs` / `SelfTest.gs`
   - （新規ファイルは「＋」→「スクリプト」。ファイル名はGASでは拡張子不要）
3. **マニフェストを表示**：⚙️「プロジェクトの設定」→「`appsscript.json` マニフェスト ファイルをエディタで表示する」をON
   → エディタに出た `appsscript.json` を、リポジトリの内容で置き換え

> 💡 手作業が面倒なら [clasp](https://github.com/google/clasp) で `gas/` フォルダをそのまま push できます。

---

## STEP 2. APIキーとメールを登録
1. ⚙️「プロジェクトの設定」→ 下の「**スクリプト プロパティ**」→「プロパティを追加」
2. 次の2つだけ手入力：

| プロパティ | 値 |
|---|---|
| `GEMINI_API_KEY` | AI Studioで発行したキー |
| `REPORT_RECIPIENTS` | レポートを受け取るメール（複数はカンマ区切り） |

---

## STEP 3. 自動セットアップを実行
1. エディタ上部の関数選択で **`setup`** を選び「実行」
2. 初回は**権限の承認**を求められる → 自分のアカウントで許可
   （「このアプリは確認されていません」→「詳細」→「（安全でないページ）に移動」でOK。自作スクリプトなので問題なし）
3. 実行ログ（表示 → ログ）に、作成された**スプレッドシート・各Doc・Driveフォルダ**のURL一覧が出ます
   → これで `SPREADSHEET_ID` などのIDは全部自動保存されています

---

## STEP 4. 動作確認（実データ不要）
関数を選んで「実行」。ログにURLが出るので開いて中身を確認します。

| 実行する関数 | 確認できること |
|---|---|
| `checkConfig` | 設定の過不足＋**Geminiの疎通** |
| `smokeTestKnowledge` | 🅱 サンプル商談→AI要約→NotebookLM母艦Doc追記→**経営レポート＆メール** |
| `demoDealProposal` | 🅰 サンプル商談→**提案書スライド**が自動生成 |
| `demoDmProposal` | 🆕 録画なしで**「マニュアル型＋提案型」提案Doc**が生成（数値つき） |

> テスト用の行はシート上で `(TEST)` 付き。`cleanupTestRows` でまとめて削除できます。

---

## STEP 5. Google Chat通知をつなぐ（任意・おすすめ）
結果を全員が見られるよう、Chatに「できました＋リンク」を流します。
1. Google Chat で通知用スペースを作る（例:「AI営業ボット」）
2. スペース名 →「アプリと統合」→「Webhook」→ 追加 → **URLをコピー**
3. スクリプトプロパティ `CHAT_WEBHOOK_URL` にそのURLを貼る
4. `testChatNotify` を実行して、Chatにテスト通知が届けばOK

（未設定でも動きます。その場合は通知だけスキップされます）

## STEP 6. 自動運用をON（これで“置くだけ”になる）
1. **`installTriggers`** を実行
2. 以後は **15分毎に `autoRunAll` が自動実行**され、新しく処理したものだけ Google Chat に通知されます：
   - 🅱 ナレッジ要約 → NotebookLM母艦Docに蓄積
   - 🅰 商談→提案書スライド
   - 🆕 DM録画/スクショ解析 → エージェント開拓提案
   - レポート（経営・DM）：毎週月曜 8:00 に更新＆通知

## STEP 6.5 承認ボタンを使う（APPROVAL_MODE / 任意）
`Config.gs` の `APPROVAL_MODE: true`（既定）だと、DMは自動で提案まで進まず、
Chatに **[✅ 提案を作成]** ボタン付きカードが届き、押した時だけ生成されます。
このボタンを動かすには **Webアプリのデプロイ**が必要です（Cloud設定は不要）。

1. Apps Script →「**デプロイ → 新しいデプロイ → ウェブアプリ**」
   - 「次のユーザーとして実行」= **自分**
   - 「アクセスできるユーザー」= **全員**
2. 発行された **ウェブアプリURL** をコピー
3. もう一度 `setup` を実行（URLとトークンを自動保存）
   - 自動で掴めない場合はスクリプトプロパティ `WEBAPP_URL` にそのURLを貼る
4. これで Chatカードのボタンが有効に。押すと承認ページが開き、提案生成/送信が実行されます。

> 全自動でよい（承認不要）なら `Config.gs` の `APPROVAL_MODE` を `false` にするだけ。
> その場合はボタンなしで、解析→提案まで自動で進みます。

**承認フロー全体**
```
Driveに録画/スクショを置く → 15分毎に自動で「解析」
   → Chatに承認カード [✅提案を作成]
   → 押すと提案書を生成し [📄開く][📧担当に送信] カードが届く
   → [送信] で担当へメール送付
```

---

## STEP 7. 本番の使い方（現場は“ファイルを置くだけ”）
| やりたいこと | 操作 |
|---|---|
| **DM営業を解析＆提案** | Driveの「**DM営業_画面録画**」フォルダに **動画 or スクショ画像** を入れるだけ |
| **複数スクショを1案件にまとめる** | フォルダ内に**案件ごとのサブフォルダ**を作り、その中に複数スクショを入れるだけ（**リネーム不要**。アップロード日時順に自動整列＋AIが会話の流れから順序を補正） |
| 会議/商談メモを貯める | GASウェブアプリにPOST、または `addKnowledge()` |
| 商談から提案書を作る | `target=deal` を付けてPOST |
| NotebookLM社内AI化 | NotebookLMで、母艦Doc「社内ナレッジ母艦」をソースに追加 |

置いたら最大15分で自動処理され、Chatに結果が届きます。Apps Scriptを開く必要はありません。
（すぐ動かしたい時だけ、スプレッドシートのメニュー「AI営業スイート → ▶ 今すぐ全部まとめて実行」）

### Web連携（任意）
1. 「**デプロイ → 新しいデプロイ → ウェブアプリ**」、アクセス「全員」で公開 → URL取得
2. n8n（`n8n/ingest-to-gas.json`）の変数 `GAS_WEBAPP_URL` にそのURLを設定
3. Notta等の文字起こしWebhookをn8nに向ければ、録音→ナレッジ/提案書まで自動化

## つまずきやすいポイント
- **`setup` で権限エラー** → 一度承認すればOK。承認画面は自作アプリなので「詳細」から進めて許可。
- **`checkConfig` でGemini失敗** → キーの貼り間違い or AI Studio側でキー有効化直後の反映待ち（数分）。
- **DM録画が取り込まれない** → ファイルが動画形式か／「DM営業_画面録画」フォルダに入っているか確認。
- **DM提案の数値が甘い** → 録画から読めない送信数・成約数は「DM案件」シートに手入力すると返信率・転換率が正確に。
- **長い録画** → まずは1案件=1録画（数分〜十数分）で。長尺はトークン増＆処理時間増。
