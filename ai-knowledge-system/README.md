# AI営業インテリジェンス スイート

2本の動画で語られている仕組みを**実際に動く形で再現**し、さらにオリジナルの
**営業DM最適解提案システム**を追加した、Google Apps Script（GAS）ベースのスイートです。

| # | システム | 元ネタ | ファイル |
|---|---------|--------|---------|
| 🅱 | 社内ナレッジ蓄積 → 経営レポート → NotebookLM社内AI | 動画①（ReZARD / 入江が開発） | `gas/Code.gs` |
| 🅰 | 商談 → 提案書(スライド)自動生成 → 送付 | 動画②（Nottaで録音→GASで5分後に提案書） | `gas/Proposal.gs` |
| 🆕 | 営業DM最適解提案（画面録画をAIが視聴 / 2つの型で提案） | あなたのオリジナル | `gas/DmCoach.gs` |

すべて **Gemini**（Google AI）で動きます。Workspaceと相性が良く、無料枠があり、
🆕 の「画面録画をそのまま解析」もマルチモーダルで実現できるためです。

---

## それぞれの仕組み

### 🅱 社内ナレッジ → 経営レポート → NotebookLM（動画①の再現）
1. 会議/商談の文字起こしを「ナレッジ」シートに投入（Webhook / n8n / Notta）
2. `processPending()` が新規行をAIで構造化（要約・課題・ネクストアクション・経営示唆・重要度）
3. NotebookLM用の**母艦Google Doc**へ自動追記 →「会社を全部知ったAI」の元データに
4. `generateWeeklyReport()` が週次で**経営レポート**を生成 → 経営者へメール & Doc

> NotebookLMには、この母艦Docを「ソース」として追加するだけ。以後はNotebookLM上で
> 「先月の失注理由は?」等を社内AIに質問できます。

### 🅰 商談 → 提案書 自動生成（動画②の再現）
1. 商談の文字起こしを「商談」シートに投入（`target=deal` でPOST、またはメニュー）
2. `processDeals()` がAIで提案書構成を作成 → **Googleスライド**を自動生成（PDFも）
3. 担当者へメール。内容を確認して顧客へ送付（自動送付にも拡張可）

### 🆕 営業DM最適解提案システム（オリジナル）
1. DM営業の**画面録画**を Drive の「DM営業_画面録画」フォルダに入れる
2. `ingestDmRecordings()` が録画を **Geminiに視聴**させ、送った文面の流れ・使った手法・
   顧客反応・結果を「DM案件」シートに構造化（文面は学習コーパス「DMメッセージ」にも蓄積）
3. `proposeForPending()` が各案件に **2つの型**で提案書(Doc)を生成
   - **① マニュアル型** … 新人が1→10まで真似できる手順書（送る順番・コピペ文面・タイミング・NG）
   - **② 提案型** … さらにアイデアが広がる切り口＋**想定 返信率／転換率**（数値つき）
   - 過去の「効果:良」の文面を学習データとして反映
4. `generateDmReport()` が**返信率・転換率**などの数値レポートを生成 → Doc & メール

## セットアップ手順

1. [Google Apps Script](https://script.google.com/) で新規プロジェクトを作成し、
   `gas/` 内の各 `.gs` と `appsscript.json` を貼り付け（マニフェスト表示はプロジェクト設定でON）。
2. **プロジェクトの設定 → スクリプトプロパティ** に最低限これだけ入れる：
   - `GEMINI_API_KEY` … [Google AI Studio](https://aistudio.google.com/apikey) で無料発行
   - `REPORT_RECIPIENTS` … レポート送付先メール（カンマ区切り）
3. `setup()` を1回実行 → スプレッドシート・各Doc・Driveフォルダを自動作成しIDを保存
   （実行ログにURL一覧が出ます）
4. `installTriggers()` を実行 → 定期実行（要約15分毎・提案書10分毎・DM取込1時間毎・レポート週次）を登録
5. Web連携するなら **デプロイ → 新しいデプロイ → ウェブアプリ**（アクセス:全員）でURL発行

以後はスプレッドシートのメニュー「**AI営業スイート**」から手動実行も可能です。

### 投入方法まとめ
| 入れたいもの | 方法 |
|---|---|
| 会議/商談メモ（ナレッジ） | GAS WebアプリへPOST、または `addKnowledge()` |
| 商談→提案書 | `target=deal` を付けてPOST、またはメニュー「商談→提案書」 |
| DM営業の画面録画 | Driveの「DM営業_画面録画」フォルダに入れるだけ |

## n8n 連携（任意）
既存の n8n を投入口として使う場合は `n8n/ingest-to-gas.json` をインポート。
Webhookで受けた内容を正規化し、GASのウェブアプリURL（n8n変数 `GAS_WEBAPP_URL`）へ転送します。
Notta等の文字起こしツールのWebhookをこのn8nに向ければ、録音→ナレッジ/提案書まで自動でつながります。

## 注意・拡張ポイント
- **APIキーは必須**：Geminiキーが無いと要約・提案・録画解析は動きません（無料枠でOK）。
- **モデル名は変わる**：Geminiのモデルは時期により提供状況が変化します（例：`gemini-2.0-flash`が
  無料枠0、`gemini-2.5-flash`が404 等）。`gas/Config.gs` の `GEMINI_MODEL` を、AI Studioで
  使えるモデルに差し替えてください（2026/07時点の動作確認モデル：`gemini-3-flash-preview`）。
- **思考型モデル対策**：思考(thinking)型モデルは思考トークンで出力上限を使い切り回答が空になることが
  あるため、既定で `GEMINI_THINKING_BUDGET: 0`（思考オフ）にしています。旧モデルに戻すなら `null` に。
- **画面録画の長さ**：長尺は解析トークンが増えます。まずは1案件=1録画（数分〜十数分）を推奨。
- **数値の精度**：録画から読み取れない送信数/成約数は「DM案件」シートで手入力すると、
  レポートの返信率・転換率がより正確になります。
- **顧客への自動送付**：🅰 はレビュー前提。自動送付にしたい場合は `processDeals()` のメール部を
  顧客宛に変更してください（誤送信防止のため既定は担当者宛）。
- LLMを Gemini 以外にしたい場合は `gas/Llm.gs` の呼び出しを差し替えるだけで全システムに反映されます。
